01 // Workflow
Collection Workflow Define → Collect → Triage → Document → Synthesize → Review
01

Define Monitored Tracks

Assign each intelligence track a standing requirement using the Collection Plan template (see below). Each track gets: named actor/geo, key identifiers, priority tier (P1/P2/P3), source list, and owner. Do not start collecting without a defined track — undirected collection produces noise. Maximum 5 active P1 tracks per analyst before quality degrades.

02

Build Persistent Collection Infrastructure

RSS aggregator (Feedly/Inoreader) with keyword alerts per track. Telegram channel watchlist per track. Google Alerts for low-frequency proper-noun terms. Ransomwatch.org bookmark for eCrime track. Do not rely on ad-hoc manual search as primary collection — every P1 track needs at least one standing automated pull. Infrastructure setup is a one-time cost; maintain and audit monthly.

03

Daily Collection Run

Realistic time budget: 30–45 min for 2–3 active tracks; 60 min for 5 tracks. Sweep aggregator priority inbox. Check Telegram watchlist. Flag items — do not read-to-understand at this stage. Apply consistent tags: [actor], [geo], [TTP], [IOC], [RUMINT], [confirmed]. Volume will be high on active conflict tracks; ruthless triage is the skill here, not completeness.

04

Triage and Corroborate

Apply the Source Tier system and Triage Decision Gate (see below). Single-source Tier 4 = log as RUMINT, do not escalate. Two independent Tier 2+ sources = escalate to actor notebook. Distinguish observation from inference explicitly. The load-bearing analytical question: "What would have to be true for this to be false?"

05

Document to Persistent Record

Push confirmed and notable RUMINT items to actor notebook. Every entry requires: date, source URL/channel, raw claim, corroboration status, confidence (H/M/L), analyst note. Log RUMINT even when unconfirmed — it establishes pattern-of-life. Silence from a previously active track is also an entry: log date, observation "no activity reported," and flag for review.

06

Weekly Delta Brief

Review accumulated entries per track. Identify: new TTPs, geographic shifts, personnel changes, capability changes, unusual quiet. Write 3–5 sentence delta brief per P1 track. Format: [what changed] / [from what baseline] / [analytical significance] / [confidence]. Do not summarize raw events — synthesize change.

07

Triggered After-Action Review

When a major event breaks (cartel leader arrested, APT campaign publicly disclosed, ceasefire announced), immediately pull all open RUMINT items for that track and run a confirmation/disconfirmation pass. Update confidence ratings. Note which prior RUMINT was validated and which was disinformation — this calibrates source fidelity. Log the trigger event as a reference anchor in the actor notebook.

05 // Cadence
Collection Cadence Daily // Weekly // Monthly // Triggered

Daily (30–60 min)

  • RSS aggregator sweep — priority inbox only
  • Priority Telegram channel check
  • ISW daily update (UA/RU + ME)
  • UK MoD release check
  • CERT-UA advisory check
  • CISA alert RSS
  • Google Alert digest
  • Ransomwatch.org scan (eCrime tracks)
  • Flag, tag, log RUMINT — do not read-to-understand

Weekly (60–90 min)

  • Run dork templates per active track
  • Vendor blog review: Mandiant, SentinelLabs, Volexity, MSTIC
  • InSight Crime + Borderland Beat cartel sweep
  • ACLED export pull for active geographies
  • Corroboration pass on open RUMINT items
  • Delta brief per P1 track (3–5 sentences)
  • Collection plan review — drop stale, add new

Monthly (2–4 hrs)

  • Full actor profile update in notebook
  • Telegram channel discovery via TGStat + TelegramDB
  • Source audit — fidelity review per source used
  • ACLED trend analysis — structured data
  • Maltego actor relationship refresh
  • Shodan/Censys infrastructure sweep for active APT tracks
  • Knowledge gap identification per track
  • Full collection plan review vs. standing requirements
  • Escalate dormant tracks to CLOSED or downgrade to P3

Triggered (on major event)

  • Pull all open RUMINT for affected track
  • Run confirmation / disconfirmation pass
  • Update confidence ratings on all related entries
  • Log which prior RUMINT was validated or disinfo
  • Log trigger event as reference anchor in notebook
  • Recalibrate source fidelity scores
  • Run priority dork sweep for new indicators
  • Assess whether track priority tier should change
⚠ ANALYST NOTES v3 //
(1) TIME BUDGET: Daily cadence assumes 2–3 active P1 tracks. Scale up to 60 min for 5 tracks. Do not attempt full coverage with less time — partial collection is worse than scoped collection.
(2) SOURCE INTEGRITY: Tier 4 sources are monitored, not trusted. Log their claims as RUMINT. Never cite single-source T4 in finished product. Graphic content is standard in cartel and conflict Telegram — establish personal exposure management protocols before expanding monitoring.
(3) OPSEC: Monitor-only posture on all adversarial channels. Do not engage, reply, or join with a traceable identity. Use separate browser profile or sandboxed VM for T4 and dark web source access.
(4) SILENCE AS DATA: When a previously active track goes quiet, log that absence. Operational silence can precede significant activity or indicate disruption. Flag for review.
(5) DORK MAINTENANCE: Date variables in dorks must be updated — stale after: dates return stale results. Review dork library monthly and roll forward all date parameters.
(6) STRUCTURED DATA PRIORITY: ACLED and Ransomwatch are the highest-fidelity aggregate sources for trend analysis. Prioritize them for pattern work over raw social and Telegram.
(7) CUSTOMIZATION: Click [ Customize ] in the nav to edit your tracks. Changes are saved to your browser’s localStorage. Use Export in the modal to back up your config.
